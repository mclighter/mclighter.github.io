Основные принципы

  • Больше внимания автоматизации
    Автоматизация для фронтенда это: grunt, bower, yooman, и многое другое.
    Для бекенда : системы непрерывной интеграции (CI), кодгенераторы. Сейчас в моей работе создание нового раздела сводится к набору пары команд в консоли. А раньше приходилось создавать кучу новых файлов вручную ... модели, контроллеры, вьюхи, все это связывать и настраивать.
  • Проектируй прежде чем реализовывать
    Прежде чем приступить к созданию приложения, необходим утвержденный прототип.
    Существует огромное количество решений для создания мокапов(прототипов):
    • https://wireframe.cc/
    • http://www.mockflow.com/
    • http://www.axure.com/
  • Следовать лучшим методикам UI-UX
    Использование готовых css фреймворков вроде Bootstrap и Foundation позволяет быстро и качественно сверстать готовую страничку. Эти фреймворки уже содержат в себе множество готовых UI элементов. Сайты на них выглядят немного шаблонно, зато чисто и гармонично. Небольшие правки css придадут сайту индивидуальность.
    • http://foundation.zurb.com/
    • http://getbootstrap.com/
    • Незабываем про новое.
